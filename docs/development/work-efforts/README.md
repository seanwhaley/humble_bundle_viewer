# Work Efforts

This folder is reserved for **active** project-management work only.

## Required structure for an active effort

Create `docs/development/work-efforts/<effort-name>/` with:

- `current-vs-to-be.md` — current state, target state, and success criteria
- `prd.md` — product requirements and scope for the effort
- `status.md` — one living status document updated over time

## Repository rule

- do not keep completed work-effort folders around once the resulting product state has been reflected in the actual docs and changelog
- do not create separate interim, summary, or final-report files for the same effort

If this folder is quiet, that is intentional: the current repository state has no active tracked work effort.
